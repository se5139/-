# V76_CHANGELOG

## v76 캡처 반려 사유 → 실제 재생성 자동 연결

v75의 캡처/OCR/수동 교정 입력과 v74 반려 대비 액션 플랜을 실제 정지형 32개·움직이는형 24개 재생성으로 연결했습니다.

### 핵심 변경
- 캡처/반려 사유 입력 → v75 보존/OCR 보조 → v74 액션 플랜 → v70 실제 세트 재생성 연결
- 사용자가 선택한 재생성 규칙을 실제 정지형/움직이는형 결과에 반영
- 재생성 정지형 32개 갤러리, 움직이는형 24개 갤러리, GIF 후보 비교 시트 생성
- Jinja2 기반 v76 HTML 리포트 생성
- 재생성 프롬프트 팩/적용 액션 플랜/manifest/SQLite 학습 DB 생성
- 생성 후 v71 QC → v72 자동보정/잠금 → v73 사용자 승인 재실행 안내

### 안전 원칙
- 기존 인기 캐릭터, 문구, 애니메이션 복제 금지
- 온라인/캡처/반려 데이터는 추상 개선 신호로만 사용
- 원본 덮어쓰기 금지
- API 키 원문 저장 금지
- 자동 제출 금지
