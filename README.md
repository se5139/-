# Kakao Emoticon Maker v100 Clean

처음부터 다시 세운 가벼운 카카오 이모티콘 제작 도구입니다.

기존 v92는 참고 자료로 남겨두고, 이 폴더는 다음 원칙으로 새로 시작합니다.

- 작게 시작합니다.
- 실행이 쉬워야 합니다.
- 최종 제출물은 PNG/GIF 중심으로 분명하게 만듭니다.
- 레거시 설치파일, 오래된 테스트 산출물, 복잡한 고급 기능은 나중에 필요한 것만 다시 붙입니다.

## 자동 검사 기준

v100 clean의 첫 기준은 앱이 생성하는 제출 후보 세트를 안전하게 확인하는 것입니다.

- 일반 정지형: PNG 32개
- 일반 움직이는: PNG 21개 + GIF 3개
- 미니 정지형: PNG 24개
- 미니 움직이는: PNG 21개 + GIF 3개
- 이미지 크기: 360 x 360 px
- 제출 ZIP: 상품 종류에 맞는 PNG/GIF만 허용
- JPG/JPEG: 미리보기 전용, 제출 ZIP 포함 금지
- PNG 용량 주의 기준: 파일당 2MB 초과
- GIF 용량 주의 기준: 파일당 500KB 초과

검사 결과는 각 결과 폴더의 `validation_report.json`에 저장됩니다.
용량 최적화 결과는 각 결과 폴더의 `optimization_report.json`에 저장됩니다.
제출 전 준비 점수는 각 결과 폴더의 `submission_readiness_report.json`에 저장됩니다.

자동 최적화 방식:

- PNG: 압축 레벨을 높이고, 필요 시 팔레트 색상 수를 단계적으로 줄입니다.
- GIF: 색상 수를 줄이고, 필요 시 프레임 수와 프레임 지속시간을 조정합니다.
- 최적화 후에도 목표 용량을 넘으면 `missed_target_count`에 기록해 사람이 확인할 수 있게 합니다.

주의: 카카오 스튜디오의 실제 제안 화면과 정책은 변경될 수 있으므로, 최종 제출 전 최신 화면에서 개수, 용량, 파일 형식을 다시 확인해야 합니다. v100은 안전한 제작 보조와 사전 검수용으로 동작합니다.

## 제출 전 준비 점수

v100은 생성 완료 후 아래 요소를 종합해 사전 점검 점수를 만듭니다.

- 규격 검사 실패/주의 항목
- 용량 최적화 목표 초과 항목
- 저작권, 상표권, 초상권, AI 정책 리스크
- 공식 출처와 주의 출처 비율
- 참고용 프로토타입 여부
- 사람 제작 증빙 메모와 러프 스케치 여부
- 사용자 직접 작성 문구의 충분성

점수는 실제 심사 통과를 보장하지 않습니다. 카카오 제출 전 사람이 최종 원본, 권리관계, 정책 적합성을 반드시 확인해야 합니다.

## 사람 제작 증빙 패키지

생성 완료 후 결과 폴더에 `evidence_package` 폴더와 `creator_evidence_package.zip`이 만들어집니다.

포함 내용:

- `evidence_summary.html`: 사람이 읽기 쉬운 증빙 요약
- `evidence_index.json`: 캐릭터, 콘셉트, 작업 모드, 준비 점수, 소스 자료 인덱스
- `rights_note_template.json`: 폰트, 브러시, 참고자료 권리 메모 템플릿
- `revision_log_template.json`: 직접 선화, 채색, 수정 이력 템플릿
- `human_origin_checklist.json`: 사람 제작 증빙 체크리스트
- `submission_readiness_report.json`: 제출 전 준비 점수
- `research_insights.json`: 조사/법적 리스크 분석
- 업로드한 러프 스케치가 있으면 `source_materials` 자료도 함께 포함

이 패키지는 “제출용 완성 증명서”가 아니라, 사람이 직접 만든 원본과 작업 과정을 정리하기 위한 보조 자료입니다.

## 실행 방법

Windows에서:

```bat
START_WINDOWS.bat
```

첫 실행 시 `.venv` 가상환경을 만들고 `Pillow`만 설치합니다. 이후부터는 같은 PC에서 훨씬 빠르게 실행됩니다.

브라우저 자동 실행 없이 서버 창만 켜려면:

```bat
RUN_SERVER_NO_BROWSER.bat
```

직접 실행:

```bat
python app.py
```

브라우저가 열리지 않으면 아래 주소를 직접 열어주세요.

```text
http://127.0.0.1:8520
```

## 웹서버 비용과 연결

v100은 내 PC에서만 실행되는 무료 로컬 웹서버를 사용합니다.

- 서버 비용 없음
- 도메인 연결 불필요
- 클라우드/호스팅 불필요
- 주소는 `http://127.0.0.1:8520`
- 다른 사람에게 공개되는 웹사이트가 아님

앱 화면의 `실행 상태 보기`에서 현재 서버 상태와 의존성을 확인할 수 있습니다.
최종 배포 ZIP을 만들기 전에는 `배포 점검` 화면에서 필수 파일과 제외 폴더를 먼저 확인하세요.

## 다른 PC 이동용 ZIP 만들기

앱 화면에서 `배포 점검`을 열고 `다른 PC 이동용 표준 ZIP 만들기` 버튼을 누르면 `release/` 폴더에 portable ZIP이 생성됩니다.

- 표준 `.zip` 형식으로 생성됩니다.
- 시간표시 ZIP과 함께 `kakao_emoticon_v100_clean_latest.zip` 고정 이름도 생성됩니다.
- GitHub에서 받는 방법은 `DOWNLOAD_LATEST_FROM_GITHUB_KO.md`에 정리되어 있습니다.
- `outputs/`, `.venv/`, `__pycache__/`, `.git/`, 로그, 기존 ZIP/EXE 산출물은 자동 제외됩니다.
- v90 레거시 도구와 개발용 폴더는 제외하고 v100 실행에 필요한 핵심 파일만 포함합니다.
- ZIP 안에는 실행 파일, README, 문서, 스크립트, 기본 메모리 파일이 포함됩니다.
- 다른 PC 실행 순서는 `QUICK_START_OTHER_PC_KO.txt`에 짧게 정리되어 있습니다.
- 배포본 변경 내용은 `RELEASE_NOTES_KO.md`에서 확인할 수 있습니다.
- 다른 PC에서는 압축을 푼 뒤 `START_WINDOWS.bat`을 실행하면 됩니다.
- 실행 전 점검이 필요하면 `VERIFY_PACKAGE.bat`을 먼저 실행하세요.
- 다른 PC 첫 실행에는 Python 3.10+와 인터넷 연결이 필요할 수 있습니다. 인터넷은 `Pillow` 설치에만 사용됩니다.
- 생성 완료 화면에서 ZIP, SHA256 체크섬, 패키지 리포트를 바로 확인할 수 있습니다.

## API 사용 여부

현재 v100은 API 키 없이 작동합니다.

- 필수 API 없음
- OpenAI API 없음
- YouTube API 없음
- 검색 API 없음

API 없이 가능한 것:

- 사용자가 입력한 URL의 제목 수집 시도
- URL 도메인 분석
- 조사 메모 기반 트렌드/법적 리스크 분석
- 기본 공식 출처 목록 기반 학습

API가 있으면 좋아지는 것:

- `YOUTUBE_API_KEY`: 유튜브 영상 제목/설명 수집 고도화
- `SEARCH_API_KEY`: 향후 검색 결과 자동 수집
- `GEMINI_API_KEY`: 향후 Gemini 분석 고도화
- `OPENAI_API_KEY`: 향후 문구/분석 고도화

v100은 키를 코드에 저장하지 않습니다. 환경변수에서만 읽고, 화면에는 마스킹해서 표시합니다.

## API 비용 방지 정책

ChatGPT와 OpenAI API는 과금/청구가 분리될 수 있으므로, v100은 API 키가 있어도 기본적으로 호출을 막습니다.

API를 사용하려면 키 환경변수와 호출 한도 환경변수를 함께 설정해야 합니다.

- `YOUTUBE_API_KEY` + `YOUTUBE_API_DAILY_CALL_LIMIT`
- `SEARCH_API_KEY` + `SEARCH_API_DAILY_CALL_LIMIT`
- `GOOGLE_CSE_ID`: Google Custom Search 검색 엔진 ID
- `GEMINI_API_KEY` + `GEMINI_API_DAILY_CALL_LIMIT`
- `OPENAI_API_KEY` + `OPENAI_API_31D_CALL_LIMIT`

한도 기준:

- YouTube/Search: 앱 자체 하루 한도
- Gemini: Pacific Time 자정 리셋 기준 하루 한도, 화면에는 한국시간 다음 회복 시각 표시
- OpenAI: 한국시간 기준 최근 31일 rolling window
- 한도가 0이거나 비어 있으면 API 호출 차단
- 사용량은 `memory/api_usage_ledger.json`에 저장
- 앱의 `API 키 안내` 화면에서 사용량과 남은 횟수를 확인 가능
- 남은 횟수가 없으면 API 호출을 중단하고 비용 없는 URL/메모 분석으로 계속 진행
- 수집 리포트에 `api_used_count`, `api_fallback_count`, `free_collection_count`를 기록
- Gemini 사용 가능 시 조사 메모를 더 정교하게 요약/분류하고, 불가능하면 로컬 분석으로 자동 fallback

Google Search 자동 수집:

- `SEARCH_API_KEY`, `SEARCH_API_DAILY_CALL_LIMIT`, `GOOGLE_CSE_ID`가 모두 있어야 작동
- 없으면 자동 검색을 건너뛰고 수동 URL/기본 출처/조사 메모 분석으로 fallback
- 수집 결과는 `search_reports`, `search_url_count`에 저장
- 수집된 URL은 출처 신뢰도 평가를 거쳐 `source_quality_summary`에 저장
- 카카오 공식/정부/저작권 기관 자료는 정책·법률 판단에 우선 반영
- 유튜브/블로그/일반 웹 자료는 트렌드 참고용으로만 분리하고 복제 금지 가드레일을 함께 기록

기존 `*_31D_CALL_LIMIT`, `*_30D_CALL_LIMIT` 환경변수도 fallback으로 읽을 수 있지만, Google 계열 새 설정은 `*_DAILY_CALL_LIMIT`를 권장합니다.

## API Fallback 정책

API 호출이 불가능하면 자동으로 무료 수집 방식으로 전환합니다.

Fallback 조건:

- API 키 없음
- 일일/31일 한도 환경변수 미설정
- 일일/31일 한도 초과
- API 요청 실패
- 네트워크 오류

Fallback 방식:

- URL 제목 수집 시도
- 도메인 분석
- 사용자가 입력한 조사 메모 분석
- API 없이 가능한 키워드/법적 리스크 분석 유지
- 프로그램은 멈추지 않고 `api_fallback_count`, `free_collection_count`를 리포트에 기록

## 기본 캐릭터 스타일

- `말랑곰`: 둥근 귀와 포근한 얼굴
- `통통토끼`: 긴 귀와 가벼운 점프 느낌
- `윙크고양이`: 삼각 귀, 수염, 윙크 표정
- `동글이`: 단순하고 말랑한 원형 캐릭터

## 감정/동작 프리셋

기본 생성물은 아래 감정 프리셋을 순환 적용합니다.

- 기쁨: 반짝이 효과
- 감사: 하트 효과
- 응원: 힘찬 폭죽 효과
- 미안: 땀방울 효과
- 애정: 큰 하트 효과
- 놀람: 느낌표/물음표 효과
- 휴식: 구름 효과
- 축하: 색종이 효과

각 감정은 효과뿐 아니라 표정 오버레이도 함께 적용합니다.

- 눈: 웃는 눈, 부드러운 눈, 하트 눈, 놀란 눈, 졸린 눈 등
- 입: 큰 미소, 작은 미소, 응원 입모양, 걱정 입, 동그란 놀람 입 등
- 볼: 밝은 볼, 하트 볼, 에너지 선, 창백한 볼, 진한 홍조 등
- 손동작: 만세, 모은 손, 주먹, 포옹, 담요 같은 보조선

적용된 표정 변형은 `expression_plan.json`의 `expression_variant`에 기록됩니다.

## 문구-감정 자동 매칭

v100은 문구 내용을 보고 감정을 먼저 정합니다.

- 감사: 고마워, 감사해, 덕분이야
- 응원: 화이팅, 응원해, 힘내, 할 수 있어
- 미안: 미안해, 실수했어, 조심할게
- 애정: 보고싶어, 좋아해, 소중해
- 놀람: 헉, 대박, 진짜?, 오잉?
- 휴식: 쉬자, 수고했어, 천천히 해
- 축하: 축하해, 최고야, 잘했어
- 기쁨: 좋아, 굿, 나이스, 행복해

매칭 이유는 `phrase_plan.json`의 `emotion_match_reason`과 `expression_plan.json`에 저장됩니다.

## 문구 자동 추천

문구를 56개 모두 입력하지 않아도 됩니다.

- 사용자가 입력한 문구는 앞쪽 슬롯에 우선 배치합니다.
- 부족한 문구는 감정 프리셋에 맞는 추천 문구로 자동 보충합니다.
- 최종 문구 배치는 결과 폴더의 `phrase_plan.json`에 저장됩니다.

## 문구 품질 검사

생성 후 `phrase_quality_report.json`에 문구 품질 점검 결과가 저장됩니다.

검사 항목:

- 긴 문구: 작은 화면에서 읽기 어려운 문구
- 중복 문구: 같은 의미가 여러 번 반복되는 문구
- 위험 표현: 혐오, 차별, 욕설, 정치/종교 공격, 브랜드/유명인 연상 키워드
- 감정 불명확: 문구만으로 감정이 잘 드러나지 않는 경우
- 자동 보충 과다: 직접 작성 문구보다 자동 문구 비율이 너무 높은 경우

품질 검사 결과는 제출 전 준비 점수에도 반영됩니다.

## 대체 문구 제안

문구 품질 검사에서 문제가 감지되면 `phrase_replacement_suggestions.json`에 안전한 대체 후보가 저장됩니다.

- 긴 문구: 짧게 줄인 후보 제안
- 위험 문구: 같은 감정의 안전 문구 후보 제안
- 중복 문구: 다른 상황/감정 문구 후보 제안
- 자동 문구 과다: 직접 작성에 참고할 추가 문구 아이디어 제안

대체 후보는 자동 참고안입니다. 최종 문구는 캐릭터 말투와 창작 의도에 맞게 사람이 직접 다듬어야 합니다.

## 대체 문구 적용 수정판

v100은 원본 결과물을 덮어쓰지 않고 `revised_phrase_variant` 폴더에 수정판 참고 세트를 별도로 생성합니다.

포함 내용:

- `revised_phrase_reference_png_gif.zip`: 대체 문구가 적용된 참고 ZIP
- `revised_phrase_plan.json`: 수정판 문구 계획
- `revised_expression_plan.json`: 수정판 표현 계획
- `revised_phrase_apply_report.json`: 어떤 문구가 어떤 후보로 바뀌었는지 기록
- `revised_phrase_refinement_report.json`: 수정판에서 남은 중복/주의 항목을 추가 개선한 기록
- `revised_phrase_quality_report.json`: 수정판 문구 품질 검사 결과
- `revised_validation_report.json`: 수정판 규격 검사 결과

원본 세트는 그대로 유지됩니다. 수정판은 안전한 후보를 빠르게 비교하기 위한 참고용입니다.

수정판은 1차 교체 후 최대 3회까지 품질을 다시 검사합니다. 중복 문구가 남으면 같은 감정의 다른 안전 후보로 다시 바꾸고, 품질 점수가 좋아지는 경우에만 갱신합니다.

## 미리보기 비교 갤러리

생성 완료 후 `preview_gallery.html`이 만들어집니다.

- 원본 미리보기와 수정판 미리보기를 나란히 표시
- 변경된 문구는 `changed` 배지로 강조
- 원본 품질 상태와 수정판 품질 상태 표시
- 전체 컷 수와 문구 변경 수 표시
- 브라우저에서 열어 빠르게 비교 검수 가능

## 최근 결과물 목록

로컬 웹서버에서 `/results` 페이지를 열면 최근 생성 결과를 볼 수 있습니다.

- 결과 폴더 이름과 생성 시간
- 캐릭터 이름과 상품 종류
- 자동 검사 상태
- 제출 전 준비 점수
- 문구 품질 점수
- 미리보기 갤러리 바로가기
- 원본 ZIP, 수정판 ZIP, 증빙 ZIP, 빌드 리포트 바로가기

`outputs` 폴더 아래 파일만 읽기 전용으로 제공합니다.

## v100 방향

1. 먼저 안정적인 최소 제작기를 완성합니다.
2. 카카오 규격 검사와 자동 보정 기능을 강화합니다.
3. 캐릭터 일관성, 표정 다양성, 반려 사유 대응 기능을 다시 붙입니다.
4. 설치파일 제작은 마지막에 붙입니다.

## 정책 안전 원칙

카카오 이모티콘 스튜디오 운영 원칙상 생성형 AI를 활용해 제작된 이모티콘 제안은 제한될 수 있습니다.

따라서 v100의 자동 생성 결과물은 기본적으로 `아이디어 참고용 프로토타입`입니다.
실제 제출은 사람이 직접 만든 원본 이미지와 작업 증빙을 기준으로 준비해야 합니다.

권장 증빙:

- 러프 스케치
- 직접 선화/채색한 레이어 원본
- PSD, CLIP, Procreate 파일
- 타임랩스 또는 단계별 캡처
- 사용한 폰트, 브러시, 참고자료 권리 메모

## 스케치 기반 제작 보조

v100은 사용자가 업로드한 러프 스케치를 캐릭터 바탕으로 사용해 참고용 이모티콘 세트를 만들 수 있습니다.

- 원형 얼굴, 몸통, 팔다리 정도의 간단한 스케치도 입력할 수 있습니다.
- 흰 배경은 최대한 투명 처리합니다.
- 스케치의 실제 선 영역을 자동 크롭하고 고정 크기/고정 중심으로 배치합니다.
- 모든 컷에서 캐릭터 중심과 비율 흔들림을 줄이기 위해 좌우 움직임을 최소화합니다.
- 정리된 스케치 레이어는 `source_materials/normalized_sketch_layer.png`에 저장됩니다.
- 스케치 분석 결과는 `sketch_consistency_report.json`에 저장됩니다.
- 콘셉트와 스타일을 바탕으로 성격, 말투, 세트 방향을 자동 설정합니다.
- 생성 결과는 기본적으로 참고용 프로토타입이며, 실제 제출 전 직접 선화/채색/수정 마감이 필요합니다.

## 자료 수집과 진화 메모리

v100은 유튜브/인터넷 URL과 조사 메모를 받아 로컬 학습 메모리를 쌓습니다.

- 자동 수집 모드는 카카오 운영 원칙, FAQ, 약관, WebP 애니메이터, 저작권 관련 기관 URL을 기본 출처로 포함합니다.
- URL에서 가능한 경우 제목만 가져옵니다.
- 각 URL의 출처 신뢰도를 평가해 공식 자료와 참고 자료를 분리합니다.
- 메모와 제목에서 추상 키워드를 분석합니다.
- 귀여움, 일상, 리액션, 위로, 개그, 심플, 모션 같은 카테고리로 정리합니다.
- 저작권, 상표권, 초상권, 생성형 AI 정책, 유해 표현 리스크를 함께 점검합니다.
- 추천 문구와 제작 방향을 `memory/evolution_memory.json`에 누적합니다.
- 타인의 캐릭터, 구도, 썸네일, 문구를 복사하지 않고 추상 경향만 참고합니다.

앱 화면에서 `자료만 수집/분석하기` 버튼을 누르면 이모티콘을 생성하지 않고 분석 결과만 저장합니다.

`반복 학습 실행하기` 버튼은 같은 자료를 여러 라운드로 재점검하고 `daily_learning_report.json`을 생성합니다.

`진화 메모리 보기`에서 누적된 학습 문구, 제작 노트, 법적 가드레일을 확인하고 중복 정리 또는 초기화할 수 있습니다.

반복해서 등장하는 문구와 카테고리는 점수가 올라가며, 다음 생성 때 더 우선적으로 반영됩니다.

생성 시 `phrase_weights`가 문구 배치에 적용되고, `build_report.json`의 `memory_weight_system`에 적용 여부와 상위 점수가 기록됩니다.

## 결과물 상세 보기

생성 후 `/results` 화면에서 각 결과물의 `상세` 버튼을 누르면 `/result?name=...` 상세 페이지로 이동합니다.

상세 페이지에서 확인할 수 있는 내용:

- 갤러리, 원본 ZIP, 수정본 ZIP, 증빙 ZIP, build report 바로가기
- 자동 검증 상태와 제출 준비 점수
- 문구 품질 점수와 문구 위험/개선 이슈
- 대체 문구 후보와 실제 수정본에 적용된 변경 내용
- 개선 루프에서 바뀐 문구 목록
- 생성된 PNG/GIF 개수와 작업 모드 정보
- 검증/제출 준비/문구 품질/수정본 상태를 색상 배지로 표시
- 자동 점검 결과를 사람이 바로 읽을 수 있는 최종 판단 박스
- 원본과 수정본 대표 JPG를 상세 페이지에서 바로 비교
